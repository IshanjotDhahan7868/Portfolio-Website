Video Automation Pipeline

My video automation pipeline converts audio files into fully packaged videos and uploads them automatically to platforms like YouTube. It uses a combination of file system watchers, AI‑generated thumbnails, audio visualizer creation and browser automation to remove all the manual steps from publishing audio content online.

Problem

Uploading music or podcast episodes as videos involves repetitive tasks: creating a thumbnail, rendering an audio waveform visualization, combining the audio and visual, filling in metadata, and finally uploading through a web interface. Doing this manually for multiple files every week wastes time and is prone to mistakes.

Solution

The automation pipeline watches a designated folder. When an MP3 file appears, it triggers a sequence: generate a thumbnail via OpenAI’s image API based on the file name, render a simple waveform visualization using MoviePy, merge audio and visual into an MP4, then open a headless browser with Selenium to upload the video. Metadata (title, description, tags) is prefilled based on file naming conventions, and the process finishes without manual intervention.

Tech Stack

  • Python scripts orchestrating the workflow
  • MoviePy and FFmpeg for audio/visual rendering
  • OpenAI API for generating thumbnails and metadata suggestions
  • Selenium WebDriver to automate browser uploads

My Role & Challenges

I architected and implemented this pipeline from scratch. I built the file watcher and processing queue, wrote Python functions to generate thumbnails and visualizers, and automated the upload process. The biggest hurdles included synchronizing asynchronous jobs, handling API errors gracefully, and keeping the Selenium automation resilient against changes in YouTube’s interface.
