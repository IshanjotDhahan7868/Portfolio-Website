FeedDoctor

FeedDoctor is an AI‑assisted tool that cleans and repairs Google Merchant Center product feeds. Retailers often struggle with incomplete data such as missing GTINs, brands, image links or product titles, which can cause their products to be disapproved. FeedDoctor automates the cleanup process and generates a ready‑to‑upload patch file.

Problem

Managing large catalogs for Google Shopping can be tedious. Product feeds frequently have missing data or invalid fields. Manually auditing hundreds of rows is error‑prone and time consuming. As a result, marketing campaigns suffer because products are rejected.

Solution

FeedDoctor lets merchants upload a CSV feed or specify a store URL. The app scans every item, cross‑checks it against public UPC/GTIN databases and suggests fixes for missing or incorrect fields. It outputs a cleaned CSV patch file with all the corrected data ready to be re‑uploaded into Google Merchant Center. The interface also highlights issues and explains why certain values are required.

Tech Stack

  • Next.js and React on the frontend
  • Serverless API routes for CSV parsing and validation
  • Node.js with OpenAI embeddings to infer missing information
  • Supabase Postgres for storing processing history

My Role & Challenges

I built FeedDoctor end‑to‑end: designing the product flow, creating the React/Next.js UI, implementing file uploads and feed parsing, integrating with external data APIs to validate GTINs, and constructing the AI model prompts that infer brand names or categories when data is missing. A key challenge was scaling CSV parsing in a serverless environment and ensuring the UI remained responsive for large feeds.
